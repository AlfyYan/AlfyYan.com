一次遇到的三个问题

Question One

diff: /../Podfile.lock: No such file or directory
diff: /Manifest.lock: No such file or directory
error: The sandbox is not in sync with the Podfile.lock. Run 'pod install' or update your CocoaPods installation.

Question Two

ld: library not found for -IQKeyboardManager

Question Three

Pods was rejected as an implicit dependency for 'libPods.a' because its architectures 'XXX' didn't contain all required architectures

　第二天要面试，想给面试官展示曾经做过的项目，于是把项目打开运行，编译却没通过，遇到第1个问题，但还好错误说明中给出了如何解决的方法——Run ‘pod install’ or update your CocoaPods installation. 关闭项目执行”pod install”。PS：此处说个小技巧，通常我们在运行”pod install/update”命令的时候会很慢,原因在于执行这两个命令的时候会升级CocoaPods的spec仓库，加个参数就会少执行这一步，速度也会相应提升：
pod install –verbose –no-repo-update
pod update –verbose –no-repo-update

　上述命令执行完之后，再次编译运行，哇咔咔。。。仍然是buil d failed o (╯□╰)o，但是这次error原因变了，变成第2、3个 错误了，谷歌ld: library not found for - IQKeyboardManager,网上有很多关于ld: library not found for...错误的解决方案，比如此处，但是我的所有第三方库都是用CocoaPods管理的，应当与链接引用没关系，搜寻各处无果。。。重新再返回xcode查看错误时，发现还有一个隐晦的warning，即上述第3个问题，(⊙o⊙)…貌似libPods.a库有问题啊，再去找谷歌，并间接寻到了万能的stackoverflow上——就是这儿，按照上面所说去修改项目，build——>run——>泪奔~~o(>_\<)o ~~讨厌的红色叉叉终于没了。。。
　 　

15-11-3遇到的问题

CocoaPods国内淘宝镜像失效

　今天在运行pod install命令的时候，突然出现command not found，自己非常确定之前已经安装过，但是既然它提示了，那么就重装一遍吧。但是执行sudo gem install cocoapods时又报错了：

ERROR:  Could not find a valid gem 'cocoapods' (>= 0), here is why:
   Unable to download data from http://ruby.taobao.org - bad response Not Found 404 (http://ruby.taobao.org/latest_specs.4.8.gz)

“not found 404”很明显是淘宝镜像失效了，而且现在使用的VPN命令行里也可以翻墙，所以决定删除原淘宝镜像(http://ruby.taobao.org)， 重新添加原来镜像地址(https://rubygems.org)。 按照当初装CocoaPods的教程, 可以知道删除镜像和添加镜像的方法分别如下:

gem sources —-remove http://ruby.taobao.org/
gem sources -a https://rubygems.org

但是执行gem remove命令时，却出现如下错误：

source http://ruby.taobao.org/ not present in cache

一直删除不掉这个淘宝镜像，最终搜索了下，在SegmentFault上找到答案，竟然是一个”/”的问题：

gem sources —-remove http://ruby.taobao.org

😥 😥 😥 😥!!
　另外镜像失效的原因是淘宝把原先的http协议镜像改为https协议镜像，所以只要将镜像地址更新为”https://ruby.taobao.org”就好了。
