前言

　Hexo是由Node.js驱动的一款快速、简单且功能强大的博客框架，尽管网上已经有很多文章介绍如何利用Hexo搭建博客，但是我们在实践的过程中仍然会遇到很多问题。谨以本篇文章记录我在使用Hexo搭建博客、写博客遇到的问题。

1号坑-Hexo安装问题

问题描述

　首先，需要明确，对于mac用户，安装Hexo之前需要install：

  • Command Line Tools

Xcode安装之后，Xcode->Preferences->Download->Command Line Tools->Install，即可安装。

  • Node.js

Node.js官网是用pkg安装包安装的，然而treehouse的这篇tutorial推荐使用Homebrew命令行安装，理由如下：

（1） 用Homebrew安装不需要给予sudo权限，这样更安全；

（2） 用安装包安装的话还要将node的可执行路径添加到系统路径(system $path下)，对于熟悉Linux或者Unix的用户，也许很容易，然而对那些对命令行不熟悉的新手来说，这是一个很痛苦的过程，而Homebrew就不需要你去配置系统路径这些。。。

（3）Homebrew是mac下的开发人员必备安装工具啊！使用安装诸如Git、Ruby等工具都将大大简化，同时卸载这些工具也很方便。

  • Git

( ⊙ o ⊙ )这步可以省略其实，因为mac自带。

以上，我确认都安装了，按照Hexo官网指示，执行命令

npm install -g hexo-cli

解决

　搜索解决问题的过程特别坎坷、痛苦，一开始谷歌关键字“npm error”，第一条使用npm安装一些包失败了的看过来（npm国内镜像介绍)就好像很相关，查看之后意识到可能是国内被墙的关系，于是按照博客上面第一种方法改镜像。

npm config set registry https://registry.npm.taobao.org
npm info underscore （如果上面配置正确这个命令会有字符串response）

重新执行安装hexo的命令。。。咕~~(╯﹏╰)b。。。还是不行。。。没办法继续搜索。。。
　网上又有说改命令变成

npm install hexo --no-optional

但是！！还是没用( ⊙ o ⊙ )啊。。。反复执行好几次仍然出现错误，就在我绝望地要按照此处同学说的卸载npm重新安装时，最后一遍执行的 nnpm install hexo --no-optional居然显示安装成功了，我现在很可惜没把当时的画面截下来。。。sigh~~

其实解决的过程稀里糊涂的、莫名其妙的，但是如果后来有同学遇到跟我一样的问题的话，可以把我上面的方法都试试，不过我隐约觉得改镜像是问题得以解决比较关键的一步。

3号坑-图片存储在Github上无法显示问题

问题描述

写到这个问题，现在回想还是一把血泪啊~~~~(>_\<)~~~~
众所周知，Markdown插入图片是要给出图片链接，但是图片存放在哪里呢？一开始我并不知道Markdown不仅可以插入网页图片，也可以插入本地图片，无论是相对路径还是绝对路径，所以开始我用的是下面的笨方法：

  2. 在终端执行’hexo d -g’将新做的改动部署到github上，并查看github上图片的链接地址：
    https://github.com/qingqinghebiancao/qingqinghebiancao.github.io/blob/master/articleImages/2015-08-10-CocoaPodsErrorFix.png

就当我重新hexo g、hexo s并满怀期待地等待结果时，结果却是页面怎么也显示不了图片。。。

解决

  1. 如果有同学采用跟我一样的笨方法——将github作为hexo的图片连接源，那么请看这篇博客，即在github的链接后面加上?raw=true：

  1. 后来，跟某大神聊到写博客插入图片的这个问题，才知道，Markdown是可以插入本地图片的，针对Hexo，可以在_config.yml文件中设置

    post_asset_folder: true

这样当我们用Hexo new "XX"新建某篇博客时，Hexo会自动为我们在_posts目录下创建一个与文章名相同的文件夹，里面可以用于存放image、css、js文件等。
