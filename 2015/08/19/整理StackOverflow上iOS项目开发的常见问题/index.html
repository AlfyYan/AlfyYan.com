基础知识点

The purpose of using IBOutlet and IBAction

IBOutlet和IBAction本质是Apple定义的宏，用来指代Interface Builder文件构建的界面上的变量和方法。比如界面上有一个UILabel控件，但是在xib对应的类文件中，你想动态改变这个label的文本，那就需要在类文件中引用界面上的这个label，IBOutlet的作用就是将两者连接起来，让其建立联系。

@property (nonatomatic, weak) IBOutlet UILabel *label;

UIView background image

【问题】 给UIView自定义背景图片
【解决】 在添加图片之前先对图片进行处理下：

UIGraphicsBeginImageContext(self.view.frame.size);
[[UIImage imageNamed:@"image.png"] drawInRect:self.view.bounds];
UIImage *image = UIGraphicsGetImageFromCurrentImageContext();
UIGraphicsEndImageContext();

self.view.backgroundColor = [UIColor colorWithPatternImage:image];

How to detect first time app launch on an iPhone

【问题】检测app是否是第一次启动
【解决】

- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions
{
    if(![[NSUserDefaults standardUserDefaults]boolForKey:@"HasLaunchedOnce"])
    {
        [[NSUserDefaults standardUserDefaults] setBool:YES forKey:@"HasLaunchedOnce"];
        [[NSUserDefaults standardDefaults] synchronize];
    }
}

CocoaPods arm64 issue

【问题】具体遇到的问题可参看我的另一篇博客：
Pods was rejected as an implicit dependency for ‘libPods.a’ because its architectures ‘armv7 armv7s’ didn’t contain all required architectures ‘armv7 armv7s arm64’
【解决】

  1. 选择Pods工程；
  2. 将Build Active Architecture Only由YES设置成NO，然后build工程。
  3. 之后可以再把NO改成原先的YES，也不会再报错了。

How to disable ARC for a single file in a project

【问题】项目开发中通常会用到第三方提供的源代码，但问题是一些比较老的开源项目可能使用的还是非ARC模式，那么我们怎么让ARC和非ARC的源文件同时存在于一个工程中呢？

【解决】Xcode项目中可以使用ARC和非ARC的混合模式。
　如果项目中使用的是非ARC模式，则为ARC模式的源文件加入fno-objc-arc标签；如果项目中使用的是ARC模式，则为非ARC模式的源文件加入fobjc-arc标签。
添加标签的方法：

  1. 打开：Target->Build Phases->Compile Sources
  2. 双击对应的.m文件
  3. 在弹出窗口中输入上面提到的标签fobjc-arc/fno-objc-arc
  4. 点击Done保存
